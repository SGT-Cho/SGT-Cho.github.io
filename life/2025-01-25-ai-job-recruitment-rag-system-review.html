AI 기반 채용정보 수집 및 RAG 분석 시스템 프로젝트 회고

January 25, 2025
12 min read
Life
RAG LLM ChromaDB LangChain 웹크롤링 Discord Bot 채용정보 AI 시스템

최근 완성한 "AI 기반 채용정보 수집 및 RAG 분석 시스템" 프로젝트를 돌아보며, 개발 과정에서 겪었던 기술적 도전과 해결 과정, 그리고 얻은 인사이트들을 정리해보고자 합니다. 이 프로젝트는 단순한 웹 크롤링을 넘어서 RAG(Retrieval-Augmented Generation) 기술과 LLM을 활용한 지능형 채용정보 검색 시스템을 구현한 것입니다.

🎯 프로젝트 개요 및 목표

취업 준비생과 이직을 고민하는 개발자들이 가장 많이 하는 고민 중 하나는 "내가 원하는 조건에 맞는 채용공고를 어떻게 효율적으로 찾을 수 있을까?"입니다. 기존의 채용 사이트들은 키워드 기반 검색에 의존하고 있어, 사용자의 진짜 의도를 파악하기 어렵다는 한계가 있었습니다.

이러한 문제점을 해결하기 위해 다음과 같은 목표를 설정했습니다:

  • 자동화된 데이터 수집: 주요 IT 기업들의 채용공고를 실시간으로 수집
  • 의미 기반 검색: 단순 키워드가 아닌 자연어로 검색 가능한 시스템
  • 실시간 알림: Discord Bot을 통한 즉시 알림 서비스
  • 개인화된 추천: 사용자 프로필 기반 맞춤형 채용정보 제공

🛠️ 기술 스택

이 프로젝트에서 사용한 주요 기술들은 다음과 같습니다:

  • Python: 메인 개발 언어
  • Selenium: 웹 크롤링
  • ChromaDB: 벡터 데이터베이스
  • LangChain: RAG 파이프라인 구축
  • OpenAI GPT: 언어 모델
  • Discord.py: 봇 개발
  • BeautifulSoup: HTML 파싱
  • FastAPI: API 서버

🕸️ 웹 크롤링: 데이터 수집의 기반

프로젝트의 첫 번째 단계는 네이버, 카카오, 삼성, LG, 쿠팡, 배달의민족 등 10개 주요 IT 기업의 채용 사이트에서 데이터를 수집하는 것이었습니다.

기술적 도전 과제들

다양한 웹사이트 구조
각 기업마다 다른 웹사이트 구조와 JavaScript 기반의 동적 콘텐츠 로딩 방식을 사용하고 있어, 일관된 크롤링 로직을 구현하기 어려웠습니다. Selenium WebDriver를 활용하여 실제 브라우저 환경을 시뮬레이션하고, 각 사이트별로 맞춤형 크롤러를 개발했습니다.

Anti-Bot 보호 시스템
대부분의 기업 사이트들이 CAPTCHA, IP 차단, Rate Limiting 등의 보호 메커니즘을 운영하고 있었습니다. User-Agent 로테이션, 랜덤 딜레이, 프록시 서버 활용, 그리고 무엇보다 윤리적 크롤링을 위한 robots.txt 준수와 적절한 요청 간격 설정을 통해 해결했습니다.

🧠 RAG 시스템: 지능형 검색의 핵심

수집된 채용공고 데이터를 단순히 키워드로 검색하는 것이 아니라, 사용자의 의도를 이해하고 의미적으로 유사한 정보를 찾아주는 것이 이 프로젝트의 핵심이었습니다.

RAG 아키텍처 구현

ChromaDB를 벡터 데이터베이스로 선택한 이유는 다음과 같습니다:

  • 🚀 빠른 유사도 검색: 임베딩 벡터 기반의 효율적인 검색
  • 🔧 쉬운 설정: Python 환경에서 간단한 설치와 설정
  • 📈 확장성: 대용량 데이터 처리 가능
  • 🔗 LangChain 호환성: 원활한 통합

LangChain을 통한 RAG 파이프라인

LangChain을 활용하여 다음과 같은 RAG 파이프라인을 구축했습니다:

  1. 문서 분할: 긴 채용공고를 의미있는 청크로 분할
  2. 임베딩 생성: OpenAI Ada-002 모델로 벡터 변환
  3. 벡터 저장: ChromaDB에 임베딩 저장
  4. 검색 수행: 사용자 질의와 유사한 문서 검색
  5. 응답 생성: LLM을 통한 자연어 응답 생성

"Python 백엔드 개발자 채용공고 찾아줘"라고 물어보면, 단순히 'Python'이나 '백엔드'라는 키워드가 포함된 공고만 찾는 것이 아니라, Django, FastAPI, Flask 등 관련 기술 스택이 포함된 공고들까지 의미적으로 연관된 결과를 제공합니다.

🤖 Discord Bot: 사용자 인터페이스

개발자들이 일상적으로 사용하는 Discord 플랫폼을 통해 서비스를 제공하기로 결정했습니다. 이는 별도의 웹사이트나 앱 설치 없이도 쉽게 접근할 수 있는 장점이 있었습니다.

주요 기능들

  • 자연어 검색: "/search 프론트엔드 개발자 React" 같은 명령어
  • 실시간 알림: 새로운 채용공고 등록 시 즉시 알림
  • 맞춤형 필터: 경력, 지역, 기술스택 기반 필터링
  • 즐겨찾기: 관심있는 채용공고 저장 및 관리

응답 시간 최적화
RAG 검색과 LLM 응답 생성에 소요되는 시간이 길어져 사용자 경험이 저하되는 문제가 있었습니다. 비동기 처리와 캐싱 시스템을 도입했습니다. 자주 검색되는 키워드에 대해서는 결과를 캐시하고, 백그라운드에서 임베딩 생성을 미리 처리하여 응답 시간을 단축했습니다.

📊 성과 분석 및 학습된 점

정량적 성과

  • 850+ 월평균 수집 공고수
  • 87% 검색 정확도
  • 2.3초 평균 응답시간
  • 95% 시스템 가동률

기술적 인사이트

1. RAG 시스템의 효과성
기존 키워드 기반 검색 대비 87%의 높은 정확도를 달성했습니다. 특히 유사 기술 스택 간의 연관성을 잘 파악하여, 'React 개발자'로 검색했을 때 'Next.js', 'TypeScript' 관련 공고들도 함께 추천하는 등의 지능적인 검색이 가능했습니다.

2. 임베딩 모델의 중요성
초기에는 무료 모델을 사용했지만, OpenAI의 Ada-002 모델로 변경한 후 검색 품질이 현저히 향상되었습니다. 한국어 채용공고의 특성을 잘 반영하는 임베딩이 핵심이었습니다.

3. 데이터 품질의 중요성
아무리 좋은 RAG 시스템이라도 입력 데이터의 품질이 떨어지면 좋은 결과를 얻을 수 없다는 것을 깨달았습니다. 크롤링한 데이터의 전처리와 정제 과정이 매우 중요했습니다.

단기 개선 사항

  • 개인화 강화: 사용자 이력서 기반 맞춤 추천
  • 실시간 시장 분석: 기술 트렌드 및 연봉 정보 제공

중장기 발전 방향

  • 확장성 개선: 더 많은 기업과 해외 채용정보 포함

  • 커뮤니티 기능: 채용 정보 공유 및 리뷰 시스템

💭 마무리 소감

이 프로젝트를 통해 RAG 시스템의 실제 활용 가능성을 직접 확인할 수 있었습니다. 단순한 검색을 넘어서 사용자의 의도를 이해하고 맥락에 맞는 정보를 제공하는 지능형 시스템을 구현할 수 있다는 것이 가장 큰 성과였습니다.

특히 LLM과 벡터 데이터베이스의 결합이 얼마나 강력한지 체감할 수 있었고, 앞으로 더 많은 도메인에서 이러한 기술을 활용한 혁신적인 서비스가 등장할 것이라는 확신을 갖게 되었습니다.
