AWS Summit LLM Tool Use Agent Bedrock AI 강연

2025년 AWS Summit에 참가하여 여러 기업들의 AWS 활용 사례와 AI 기술 동향을 살펴볼 수 있는 귀중한 기회를 가졌다. 특히 인상 깊었던 강연 중 하나는 LLM의 한계를 극복하는 Tool Use와 Agent 개념, 그리고 Amazon Bedrock을 활용한 구현 방법에 관한 세션이었다.

🔧 LLM의 한계와 Tool Use의 필요성

강연은 현재 LLM(대규모 언어 모델)이 가진 근본적인 한계점들을 짚어보는 것으로 시작되었다. 아무리 방대한 데이터로 학습된 모델이라도 다음과 같은 제약이 존재한다:

  • 수학 계산의 한계: 세 자릿수 이상의 복잡한 계산에서 정확도가 현저히 떨어짐
  • 실시간 정보 접근 불가: "현재 서울 날씨"와 같은 실시간 정보는 알 수 없음
  • 학습 데이터의 시점 제약: 모델이 학습된 시점 이후의 정보는 반영되지 않음

이러한 한계를 극복하기 위해 등장한 개념이 바로 Tool Use다. 2020년 발표된 MRKL 시스템 논문에서 처음 제안된 이 아이디어는 모델이 외부 세계와 상호작용할 수 있는 방법을 제시했다.

⚙️ Tool Use의 동작 원리

Tool Use의 핵심은 모델이 정적인 지식만 사용하는 것이 아니라 실시간 정보를 기반으로 실행 능력까지 갖추게 하는 것이다. 일반적인 동작 과정은 다음과 같다:

  1. 툴 정의: 모델이 선택할 수 있는 외부 툴(함수) 목록을 미리 정의
  2. 툴 선택: LLM이 사용자의 질문에 답변하기 위해 필요한 툴을 스스로 선택
  3. 툴 실행: 선택한 외부 툴(함수)을 실행하고 그 결과를 다시 LLM에게 전달
  4. 답변 생성: LLM은 결과값을 바탕으로 사용자의 질문에 대한 최종 답변을 생성

여기서 중요한 점은 모델은 함수 호출만 제안하는 것이고, 실제 실행은 외부 시스템이 담당한다는 것이다. 이를 통해 웹 검색, 데이터베이스 조회, 시스템 명령 실행 등 다양한 작업을 수행할 수 있다.

🤖 Agent: 자율적인 업무 수행 시스템

Agent는 Tool Use를 한 단계 더 발전시킨 개념이다. 단순히 사용자의 질문에 답변하는 것을 넘어, 가상 직원처럼 복잡한 업무를 자율적으로 수행하는 시스템이다.

Agent의 핵심 특징은 다음과 같다:

  • 자율성(Autonomy): 해야 할 작업을 스스로 계획하고 실행
  • 외부 영향력(External Influence): 외부 환경과 상호작용하며 실제 업무 수행
  • 다단계 처리: 복잡한 작업을 여러 단계로 나누어 순차적으로 처리

과거에는 모델이 사용할 수 있는 툴의 수가 제한적이고 단순한 작업만 처리 가능했지만, 모델이 점점 똑똑해짐에 따라 수십, 수백 개의 툴을 안정적으로 사용하여 매우 복잡하고 다단계적인 작업을 처리할 수 있게 되었다.

☁️ Amazon Bedrock을 통한 구현

Amazon Bedrock은 이러한 Tool Use와 Agent 기능을 쉽게 구현할 수 있는 다양한 기능을 제공한다:

1. Converse API를 통한 Tool Use

복잡한 구현 과정 없이 표준화된 문법으로 Tool Use를 편리하게 사용할 수 있다.

2. Bedrock Agent의 구조

Bedrock Agent 시스템은 세 가지 핵심 요소로 구성된다:

  • 에이전트 자체: 오케스트레이터 역할을 수행
  • Bedrock 모델: 실제 추론과 계획 수립을 담당
  • 툴: 액션 그룹과 지식 기반으로 구성

3. 액션 그룹과 지식 기반

  • 액션 그룹(Action Group): 목적에 맞게 구현된 함수의 모음
    • Lambda 방식: 에이전트가 자동 호출
    • Caller Control 방식: 클라이언트에게 제어권 넘김
  • 지식 기반(Knowledge Base): 자연어 기반 정보 검색 기능 제공

💼 실제 데모: S&P 500 연간 변동성 계산

강연에서 가장 인상 깊었던 부분은 실제 데모였다. 금융 회사의 신입 개발자가 S&P 500의 연간 변동성을 계산해야 하는 복잡한 과제를 받았을 때의 시나리오였다.

문제상황: 변동성 정의나 계산 방법을 모르는 개발자가 복잡한 금융 계산을 수행해야 함

해결과정:

  1. 코드 인터프리터를 활용한 날짜 계산 (오늘 날짜, 32일 전 날짜 계산)
  2. 액션 그룹 함수를 통한 주식 데이터 수집
  3. 다시 코드 인터프리터를 사용한 변동성 계산 (라이브러리 활용, 데이터 반영)
  4. 계산 결과를 바탕으로 한 최종 답변 생성 및 해석 제공

이 과정에서 개발자는 직접 복잡한 계산 로직이나 코드를 작성할 필요 없이, 에이전트가 스스로 계획하고 필요한 툴을 활용하여 문제를 해결하는 것을 확인할 수 있었다.

🎯 핵심 인사이트

이번 강연을 통해 얻은 핵심 인사이트는 다음과 같다:

  • Tool Use는 필요한 툴을 호출하여 작업을 빠르고 정확하게 처리하는 방식
  • Agent는 계획을 세우고 필요한 툴을 반복적으로 사용하며 복잡한 작업을 자동화하는 방식
  • 도메인 지식이 Tool Use 성공의 핵심 요소
  • Amazon Bedrock을 통해 이러한 고급 기능들을 쉽게 구현할 수 있음

🔮 미래 전망

AI 에이전트 기술의 발전은 단순한 질답 시스템을 넘어 실제 업무를 수행하는 가상 직원 수준까지 발전하고 있다. 특히 AWS Bedrock과 같은 클라우드 플랫폼의 지원으로 이러한 기술들이 더욱 접근 가능해지고 있다는 점이 인상적이었다.

앞으로 AI 엔지니어로서 이러한 Tool Use와 Agent 기술을 실제 프로젝트에 적용해보고, 더 나은 사용자 경험을 제공하는 AI 시스템을 구축해나가고 싶다는 생각이 들었다.

AWS Summit 2025는 최신 AI 기술 동향을 파악하고, 실무에 적용할 수 있는 구체적인 방법론을 학습할 수 있는 매우 유익한 경험이었다. 특히 이론적인 설명뿐만 아니라 실제 데모를 통해 기술의 실용성을 직접 확인할 수 있어서 더욱 의미 있었다.
